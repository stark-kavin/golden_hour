The Bus Factor

Strategies for Knowledge Distribution &
Risk Mitigation

Metric Definition

What is it?

"The minimum number of team members that have to disappear before a project stalls."

The Lottery Scenario

If your lead engineer wins the lottery and moves to Fiji, does the server crash?

The Goal

We aim for a HIGH number. Redundancy is resilience.

Critical Risk

The "Hero" Bottleneck

Why a Bus Factor of 1 is dangerous.

Knowledge Silos

When only one person holds the keys, the code, or the context, the entire team is held hostage by their availability.

Burnout Risk

The "Hero" can never truly disconnect. They can't take sick days or vacations without anxiety.

Single Point of Failure

If the Hero leaves, the feature dies. The legacy code becomes a "black box" that no one dares to touch.

The "Vacation Test"

30-Day Offline Challenge

1

Could the team deploy to production?

2

Could the team fix a critical bug?

3

Do they know where the API keys are?

If you answered "No", we have a problem.
Remediation Strategy

Increasing the Factor

Micro-Docs

Stop writing massive wikis. A simple README.md or a 2-minute screen recording is infinitely better than nothing.

Pair Programming

Don't code in a cave. Share context while you work. Rotate tasks so everyone touches the "scary" parts of the codebase.

Access Management

Use password managers like 1Password or LastPass for shared accounts. Ensure admin access isn't tied to a single user.

Key Takeaway

Make yourself replaceable,
so you can be promotable.

Immediate Action Item

Pick one process.

Document it today.
